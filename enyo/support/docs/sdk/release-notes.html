Release Notes - webOS SDK 3.0 Beta 1

webOS 3.0 SDK Beta 1 is a major update, providing developers with a wealth of new features across the software platform, detailed below.

Tools and Installers

palm-package

In the 3.0 SDK, the palm-package tool has been updated to properly package Enyo apps so they can be installed onto the Palm Emulator (using palm-install) and launched (using palm-launch or the webOS application launcher).

It is not necessary to include a copy of the Enyo framework inside the application, nor is it necessary to modify the script tag that includes "enyo.js", since palm-package will re-write this (to work in the emulator or device) when it builds the application package, as long as the script tag ends with "0.8/framework/enyo.js".

This means that you can develop on the desktop using a relative (or absolute) path to the Enyo framework, and palm-package will update the tag when it creates the package.

Enyo Framework

The Enyo framework has been updated from version 0.6 to 0.8.

Gestures and Events

  • This should be a transparent change, but the way in which mouse events are handled has changed considerably. This should provide for better handling of clicks across element boundaries, and it provides basic drag and drop handling.

    • New standard Enyo events for: ondragstart, ondrag, ondragfinish (fire on the control that is being dragged).

    • New standard Enyo events for: ondragover, ondragout, ondrop (fire on the control that is being dragged over).

    See support/examples/Drag for examples.

  • It is now possible to prevent focus changes by implementing onmousedown and calling inEvent.preventDefault(). This can be used to prevent the virtual keyboard from hiding.

  • Users of the DOM attribute "enyo-pass-events" should be aware that it has been removed in favor of an explicit control flag, "requiresDomMousedown".

  • Removed 100ms delay in processing click event.

Cross-App UI

An enyo.CrossAppUI component has been added to allow x-app content. The content is displayed inside an <IFRAME> and applications are responsible for sizing and positioning the UI. The app property specifies the id of the app to get content from; path is the HTML document in that app to use; and params are the data to send to the app. The onResult event fires when the x-app content sends a result to your application. 

{kind: "CrossAppUI", params:{url:"http://www.palm.com"}, app:"com.palm.app.enyo-browser",
  path: "index.html", onResult: "handleResult"}

The enyo.CrossAppResult component can be used to send a result from the x-app content back to the host application.

Globalization (g11n) and Internationalization (i18n)

The Globalization library "g11n" is now part of Enyo.

As they are now handled by g11n, enyo.Date, enyo.addOnStart, and enyo.getLocale have been removed.

The core of g11n is automatically built into Enyo; however the address, name, and phone number submodules must be loaded separately. This can be done with the following code: 

enyo.depends(
  "$enyo/g11n/name/",
  "$enyo/g11n/address/",
  "$enyo/g11n/phone/"
);

UI Controls

  • SlidingGroup is officially deprecated in favor of SlidingPane, a proper Pane kind.

    • SlidingPane supports the same API as Pane, so instead of setSelected and getSelected, use selectView and getView.

    • SlidingPane should contain SlidingView controls (SlidingView replaces Sliding); SlidingView is the defaultKind of SlidingPane.

    • SlidingPane replaces the "wideLayout" with "multiView" and "wideWidth" with "multiViewMinWidth".

  • The animation has been redone to use JavaScript instead of CSS, but the change should be mostly transparent. For example, it's possible to specify a custom easing function for the animation.

  • Popups should now reposition themselves when the window is resized.

Windows

enyo.window.setWindowParams() is now an asynchronous call, and the params are limited to valid JSON structures (no cycles on the object graph, no functions, and so on).

WebViews

  • Added padding around smart zoom rect.
  • Keep user pan/zoom during load.
  • Respect min/max zoom from meta tag.
  • Set default layout width to 1024.
  • Disable pinchzoom by meta tag.

New Folder Structure

The Enyo folder structure has been changed. This should be mostly transparent to apps, with the following exceptions:

  • enyo.js is now in the 0.8/framework folder. Please update the script path, e.g., 

    <script src="../../enyo/0.8/framework/enyo.js" type="text/javascript"></script>

  • In depends files, update loading of theme packages to use the $palm alias rather than $enyo, e.g., 

    "$palm/themes/Gemstone-default/",

API Changes

  • Removed enyo.dom.fetchNodeOffset. Replaced with a simplified enyo.dom.calcNodeOffset, which is protected and calculates offset using only offsetTop/Left and does not adjust for webkitTransform or scroll position.

  • In enyo.ManagedDomBuilder, added public method getOffset, which calculates control offset and adjusts for any offset that controls specify (e.g., la webkitTransform, scroll position).

  • openNearNode(inNode, inOffset) has been removed; instead, use openAtControl(inControl, inOffset).

  • openAroundNode(inNode) has been removed; instead, use openAroundControl(inControl).

  • The API has been adjusted to no longer use l, t, r, and b properties. Instead, left, top, right, and bottom are used. For example, in openAt(inRect), inRect may contain left, top, right, or bottom properties.

  • Added code to preserve max-height/width sizes set by the user; note that we set max height/width to ensure popup is not larger than the viewport.

  • Added enyo.fetchAppRootPath.

  • Added string util for escaping and removing HTML.

Plugin Development Kit

New Hardware

There are additional hardware identifiers that can be returned by: 

#define HARDWARE_VEER       301
#define HARDWARE_PRE_2      401
#define HARDWARE_PRE_3      501
#define HARDWARE_TOUCHPAD   601

Virtual Keyboard 

PDL_Err PDL_SetKeyboardState(PDL_bool bVisible);

If your application takes keyboard input, you need to be aware that with the HP Touchpad, we're starting to produce devices that do not have a physical keyboard. On these systems, you use PDL_SetKeyboardState to turn the on-screen keyboard on and off. When on-screen, the keyboard will send keyup/down SDL events. The keyboard can be dismissed by the user, so you should either provide a way for the user to toggle the keyboard on/off or always enable it when your app wants keyboard input.

Compass 

typedef struct
{
  int confidence;
  double magneticBearing;
  double trueBearing;
} PDL_Compass; 

PDL_Err PDL_EnableCompass(PDL_bool activate);
PDL_Err PDL_GetCompass(PDL_Compass *compass);

These APIs are for devices that include a compass sensor, like the HP Pre 3. Since the compass uses power, you should only enable it when it is needed. When it's turned on, you can either poll for its current setting using PDL_GetCompass or wait for PDL_COMPASS events to be sent through the SDL event loop.

Emulator

Starting with the webOS 3.0 emulator, you are able to run x86-compiled PDK code in either full-screen or hybrid mode. Code using the SDL APIs is supported, but OpenGL ES is not yet functional in emulation. You need a separate x86 toolchain to build PDK code. On Windows, you can download the current CodeSourcery G++ Lite cross-compiler from http://www.codesourcery.com/sgpp/lite/ia32/portal/release1567. We have seen problems with these tools when hosted on 64-bit Windows.

Known Issues and Limitations

Tools and Installers

VirtualBox

  • If you are installing and using VirtualBox for the very first time, please launch the VirtualBox application first before starting the Palm Emulator. Or, if you have already tried to start the Palm Emulator and found that it won't start, type the following command from a command shell: 

    VBoxManage setextradata global "GUI/UpdateDate" "never"

Emulator

  • When installing an application into the 3.0.0.611 Palm Emulator with palm-install, the application's icon may not appear in the launcher until the system manager (or emulator) is restarted. The app can still be launched immediately with palm-launch.

    To restart the system manager, type the following in a host command shell: 

    novacom run file:///sbin/stop LunaSysMgr && novacom run file:///sbin/start LunaSysMgr

Enyo

Cross-App UI

  • It's currently possible for JavaScript code to affect things on the other side of the iframe boundary.

UI Controls

  • While SlidingPane supports the Pane API, lazy views have not been tested and may not be well supported in this release.
