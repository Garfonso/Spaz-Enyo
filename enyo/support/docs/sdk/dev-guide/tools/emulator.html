Emulator

Overview

The HP webOS emulator emulates a webOS device on a Linux, Mac, or Windows computer. It is built on VirtualBox, virtual machine software that is free from Oracle. If you installed the SDK, you already have the emulator on your computer.


Figure: HP webOS Platform Developer Toolset 1 (emulator or device support)

The emulator includes a radio simulator that lets you simulate phone calls and GPS fixes. For more info, see Radio Simulator.

Currently, the emulator does not support the following:

  • Accelerometer live data (does support orientation and shaking events)
  • Audio
  • Bluetooth (UI and any interface-specific APIs do not work in the emulator)
  • Camera
  • Gesture area
  • Multi-touch
  • Sound
  • Video
  • Wi-Fi (UI and any interface-specific APIs do not work in the emulator, but network connectivity works by using the desktop connection)

Starting the Emulator

Follow the instructions for your system:

  • Linux: At the command prompt, type palm-emulator.

  • Mac: In the Applications folder, double-click the Emulator icon.

  • Windows: Select Start > All Programs > Palm > SDK > Emulator.

Running Applications on the Emulator

If you are using Eclipse with the HP webOS plug-in, you can run an application on the emulator by selecting Run > Run Configurations..., then selecting "Palm Emulator" for the target of your application name. On subsequent activations, select Run > Run As... > Mojo Application. Eclipse automatically packages, installs, and launches the application. With Eclipse, there is no need to uninstall the application before re-running it.

If you are using the command line tools, run the application at the command prompt by using the webOS tools included with the SDK.

Notes:

  • These instructions use the "Hello, World!" example. Remember to make changes as needed for actual applications.
  • The "workspace directory" contains all of the application directories.
  • Changes made during development and debugging may not appear unless you first delete the previous version, and then repackage and reinstall the updated application.
  • Installing a new version of an application does not remove any source files that are not present in the new version. Therefore, during testing, it is useful to delete the old version of an application before installing a new version.

Package and install the application on the emulator

  1. Start the emulator.

  2. Open a Command Prompt window, and then navigate to the workspace directory.

  3. Use the palm-package command as follows to package the directory containing the application: 

    palm-package HelloWorld

  4. Use the palm-install command as follows to install the resulting .ipk file on the emulator. (Note that if you do not have the emulator running, but do have a webOS device plugged into your computer's USB port, this will install the package to the device if you have enabled Developer Mode. You will find your app's icon at the bottom of the first launcher menu. When plugging the device into the computer to load an app, select the "Just Charge" option on the device.) 

    palm-install com.mystuff.hello_1.0.0_all.ipk

Launch the application at the command prompt

  1. Start the emulator, if it is not already started.

  2. Open a Command Prompt window, and then navigate to the workspace directory.

  3. Use the palm-launch command as follows: 

    palm-launch com.mystuff.hello

Close the application at the command prompt

  1. Start the emulator, if it is not already started.

  2. Open a Command Prompt window, and then navigate to the workspace directory.

  3. Use the palm-launch command as follows: 

    palm-launch -c com.mystuff.hello

Delete the application from the emulator

  1. Start the emulator, if it is not already started.

  2. Open a Command Prompt window, and then navigate to the workspace directory.

  3. Use the palm-install command as follows: 

    palm-install -r com.mystuff.hello

Debugging an Application

There are two ways to debug an application in the emulator:

  • Use the Debugger
  • Use the console output of the emulator

Set up the console output on Windows

  1. Open PuTTY from the Palm > SDK program group.

  2. Enter localhost as the Host Name and 5522 as the port.

  3. If a security alert dialog box appears, press Yes.

  4. For the login name, enter root, and then press Enter.

  5. For the password, press Enter (the password is blank).

  6. At the emulator command prompt root@qemux86:/var/home/root#, enter the following command: 

    log com.yourdomain.your-app

Set up the console output on Mac or Linux

  1. Open a terminal.

  2. Run the following command: novaterm.

  3. At the emulator command prompt root@qemux86:/#, enter the following command: 

    log com.yourdomain.your-app

For more info about generating logs from your application, see Logging and palm-log.

Keyboard Shortcuts for the Emulator

The following table lists the keyboard shortcuts supported in the emulator.

Key Description
ALT (Windows and Linux) or OPTION (Mac) Option key.
END Opens and closes the launcher.
ESC Performs the back gesture (or swipe back).
Home Minimizes and maximizes the card.
LEFT/RIGHT ARROW Switches the application left or right in the Card view.
Sym (symbol) key Control key.

Exiting the Emulator

To exit the emulator, select Machine > Close > Power off the machine > OK.

Note:

If you choose a different option, you cannot install applications on the emulator the next time you start it.

Performing a Hard Reset

Follow the instructions for your system:

  • Linux: At the command prompt, type the following: 

    palm-emulator --reset

  • Mac: Open a terminal, and type the following: 

    /Applications/Palm\ Emulator.app/Contents/MacOS/palm-emulator --reset

  • Windows: At the command prompt, type the following: 

    \\Program Files\Palm\SDK\bin\palm-emulator.bat --reset

Running VirtualBox

Follow the instructions for your system:

  • Linux: At the command prompt, type VirtualBox.

  • Mac: In the Applications folder, double-click the VirtualBox icon.

  • Windows: Select Start > All Programs > Sun xVM VirtualBox > VirtualBox.

Keyboard Capture

By default, VirtualBox captures the keyboard into a virtual machine. On the Mac, this can interfere with using the left command key to switch among applications with command-tab. If you run into this problem, try the following:

  1. Exit the emulator.
  2. Open VirtualBox.
  3. On the VirtualBox menu, select Preferences.
  4. Switch to the Input section.
  5. Press the right command key to switch the Host Key, and then turn off Auto Capture Keyboard.
  6. Press OK, and then close VirtualBox.

Accelerometer Events

The emulator supports certain accelerometer events through the use of function keys.

Key Description
F5 Simulate shaking
F6 Simulate "up" (12:00 or normal) orientation
F7 Simulate "down" (6:00) orientation
F8 Simulate "right" (3:00) orientation
F9 Simulate "left" (9:00) orientation

Note that the orientation keys will only be active if Window Orientation has been set to "free".

New Beta feature: Emulator Host Mode

The 1.4.1 release of the SDK includes a beta version of Emulator Host Mode. This mode allows you to inspect your application using a desktop browser (Chrome or Safari). To accomplish this, the emulator now contains its own web server that renders your application on port 8080.

After the emulator is started and you've set up an SSH tunnel between the emulator and the desktop (described below), you can launch a browser on your desktop and render any applications in your browser directly! (Note that if using Chrome you'll need to disable the popup blocker.)

Setting up an SSH tunnel

To run in Emulator Host Mode, you need to set up an SSH tunnel between the emulator and your desktop. The procedures differs between platforms. The following sections explain how to set up the tunnel.

Emulator Host Mode on Mac or Linux

Open a shell and start an SSH tunnel by executing the following command: 

ssh -p 5522 -L 5581:localhost:8080 root@localhost

When prompted for a password, hit "Enter". Leave this terminal window open for as long as you need to use the Emulator Host Mode.

Emulator Host Mode on Windows

There are two options for running Emulator Host Mode under Windows: Cygwin and Tunnelier. Each will set up an SSH tunnel between your host and the emulator.

The following steps will take you through the process of installing and setting up Cygwin:

  1. Download the Cygwin setup executable.
  2. Double-click on the downloaded setup.exe file to launch the Cygwin Net Release Setup program.
  3. Click on "Next" to continue with the installation.
  4. Select "Install from Internet" and click "Next".
  5. Choose an installation directory, or accept the default, and click "Next".
  6. Choose a local package directory, or accept the default, and click "Next".
  7. Select your Internet connection type and click "Next"
  8. Pick a mirror download site and click "Next". The download will begin.
  9. In the "Cygwin Setup - Select Packages" window, under the "All", then "Net" categories, select "openssh". The program will also select some dependencies automatically. (The default package selection does not include openssh.)
  10. Click on "Next" to start the installation.
  11. After installation is complete, double click on the "Cygwin" icon on desktop or in Programs Folder to launch the Cygwin shell.

  12. Start an ssh tunnel in the shell by executing the following command (assuming you have started the webOS emulator under Virtual Box already.): 

    ssh -p 5522 -L 5581:localhost:8080 root@localhost

The following steps will take you through the process of installing and setting up Tunnelier:

  1. Download the Tunnelier SSH client.
  2. Double-click on the downloaded Tunnelier-Inst.exe to launch the installer.
  3. Click in the check box to agree to the license terms and click on "Install". Once the installation is complete, the application will launch.
  4. Under the "Login" tab, set the following options:
    • Host: localhost
    • Port: 5522
    • Username: root
  5. Under the "C2S Fwding" tab, add a new forwarding entry by clicking on Add. Set the following options:
    • Listen Interface: 127.0.0.1
    • Listen Port: 5581
    • Dest Port: 8080
  6. Click on "Login" to login. (This assumes you have started the webOS emulator under Virtual Box already.)
  7. The tunnel is now up.

Using Emulator Host Mode

Open your web browser, and enter one of the following addresses:

  • http://localhost:5581/PalmApps for webOS internal applications.
  • http://localhost:5581/Apps for 3rd party applications.
